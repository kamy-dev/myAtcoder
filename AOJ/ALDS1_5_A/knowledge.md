## Point
- 再帰関数を用いると、ある問題を2つ以上の小さい問題に分割して、再帰関数によりそれぞれの部分問題の会を求め、それらの結果を統合することで、元の問題の解を求められる。
    - これを**分割統治法**と呼ぶ

### 分割統治法

```bash
[手順]
1. 与えられた問題を、部分問題に「分割」する (Divide)
2. 部分問題を再帰的に解く (Solve)
3. 得られた部分問題の解を「統合」して、元の問題を解く (Conquer)
```

## 本問題の思考履歴

結論から言うと 「途中の sum も、最終的にちゃんと A_set に入る」 のですが、なぜそうなるのか、しっかり追ってみましょう。

勘違いの正体
おそらく、こう思っていませんか?

「sum = 1（A[0] だけ使った状態）も和として作れるはずなのに、i == n のときしか add してないから、入らないんじゃない?」

これは「途中の状態」と「葉っぱ（末端）の状態」を混同していることから来ています。実は、「A[0] だけ使った」という状態も、最終的には葉っぱに到達します。

A = [1, 5, 7] で具体的に追ってみる
「A[0]=1 だけ使う」というパターンが、どこで A_set に追加されるか追ってみてください。

```
A_sum_set(0, 0) を呼ぶ
  ↓ A[0]=1 を「使う」枝に進む
  A_sum_set(1, 1)        ← sum=1（A[0] を使った状態）
    ↓ A[1]=5 を「使わない」枝に進む
    A_sum_set(2, 1)       ← sum=1 のまま（A[1] は使わない）
      ↓ A[2]=7 を「使わない」枝に進む
      A_sum_set(3, 1)     ← sum=1 のまま（A[2] も使わない）
        ↓ i == n になった！
        A_set.add(1)  ✅ ここで追加される
```

つまり、「A[0] だけ使う」というパターンは、

「A[0] を使う」 → 「A[1] を使わない」 → 「A[2] を使わない」

という決断の連続として表現され、**最後の決断が終わった時点（葉っぱ）**で A_set に追加されるんです。

なぜ「葉っぱでしか add しない」で全パターンが網羅されるのか
ポイントは:

「使わない」も 1 つの選択肢として、ちゃんと再帰の中で扱われている

ということです。

```python
A_sum_set(i + 1, sum + A[i])  # A[i]を使う
A_sum_set(i + 1, sum)          # A[i]を使わない ← これが重要！
```
「使わない」も 1 つの決断として最後まで進んでいくので、「A[0] だけ使って残りは使わない」というパターンも、必ず葉っぱに到達します。

全 8 パターンを列挙してみる
A = [1, 5, 7] の場合の全パターン（使う:○、使わない:×）:

```
A[0]=1	A[1]=5	A[2]=7	葉での sum
○	○	○	13
○	○	×	6
○	×	○	8
○	×	×	1 ← 「A[0] だけ使う」もここで記録される
×	○	○	12
×	○	×	5
×	×	○	7
×	×	×	0
```

ぜんぶの組み合わせ（2^3 = 8 パターン）が、それぞれ葉っぱで add されることがわかります。

```
図で全体像
                A_sum_set(0, 0)
                ↓ 使う          ↓ 使わない
        A_sum_set(1, 1)        A_sum_set(1, 0)
        ↓使う ↓使わない        ↓使う ↓使わない
    (1,6)  (1,1)... 葉っぱまで全部進む
```

どの経路でも、必ず i==n まで進んでから add される
→ 経路の数 = 葉っぱの数 = 2^n パターン全部

**「途中で add しない」のではなく、「途中の状態も、いずれ葉っぱとして到達する」**というのが正しい理解です。

別の見方: 「使わない」=「sum を変えずに次へ」
もう 1 つ別の角度から見ると、「使わない」枝は 「sum を変えないまま i だけ進める」 という動きです。だから「A[0] だけ使う」場合は、

sum=1 のまま、i=1, i=2, i=3 と進んでいき、最後に sum=1 として記録される
という動きになります。途中で記録する必要はなくて、葉っぱに到達したときの sum がそのまま記録されるので OK なんです。

まとめ
| あなたの疑問 | 実際 |
|---|---|
| 途中の sum は記録されない? | 葉っぱで全部記録されるので大丈夫 |
| なぜ? | 「使わない」も再帰の枝として最後まで進むから|
| 結果として | 2^n 通り全部の sum が A_set に入る |

この理解はとても重要なので、引っかかったのは良い学びです！再帰は「葉っぱで結果を取る」という構造を持つことが多いので、この感覚は今後も役立ちます 👍

実際に動かして確かめてみるのも良い理解になります:

```python
n = 3
A = [1, 5, 7]
A_set = set()

def A_sum_set(i, s):
    if i == n:
        A_set.add(s)
        return
    A_sum_set(i + 1, s + A[i])
    A_sum_set(i + 1, s)

A_sum_set(0, 0)
print(sorted(A_set))  # → [0, 1, 5, 6, 7, 8, 12, 13]
```
