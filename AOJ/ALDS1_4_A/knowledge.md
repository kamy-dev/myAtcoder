## Input

１行目にn、２行目に S を表す n 個の整数、３行目に q、４行目に T を表す q 個の整数が与えられます。

## Output

C を１行に出力してください。

## Point

### Linear Search とは
探索したい数値が、ある配列内に存在するかを先頭から末尾まで順番に探索する手法
時間計算量は**O(n)** (番兵を利用した場合、O(qn)となる。これはforループの継続条件の判定が不要になるため、定数倍高速になる。)

#### 番兵について
実際の実行速度が約2倍速くなる

  オーダー記法（O(n)）は定数倍を無視しますが、実際のプログラムでは定数倍は効いてきます。

  例: n = 1,000,000 の配列を探索

  番兵なし: 条件判定 約2,000,000回（2つ × 100万）
  番兵あり: 条件判定 約1,000,000回（1つ × 100万）

  ただし正直なところ、現代のプログラミングでは番兵の恩恵は小さいです。CPUが高速なので条件判定1つ減らしても体感差はほとんどありません。競技プログラミングでギリギリの制限時間に間に合わせたい、といった場面で稀に役立
  つ程度です。

  概念として知っておく価値はありますが、実用上は二分探索やハッシュなどアルゴリズム自体を変える方がはるかに効果的です。
